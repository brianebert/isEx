Hi, I'm Brian. You're reading this page out of ipfs. They can tell you about themselves better than I can, amd I hope you'll browse their site. This page owes a lot to their videos, which made me think more about the effects of what I do. Further reading on how to use ipfs on these pages is available from menu item using ipfs here.

When you add a file to ipfs that data is assigned a unique address derived from its content. Any ipfs node can read the file added by requesting the contents of the ipfs address. The data moves in torrents. Any change to the content and the address changes, so recording the address renders the contents. immutable. You can find a list of ipfs gateway nodes here.

I only use a small portion of the of ipfs api. One notable part of ipfs I don't use is the name system, ipns. I read early on that the ipns implementation was experiencing problems, and I already knew how to post key/value pairs on the stellar blockchain. Stellar closes blocks in seconds, so I used stellar data entires to relate human readable names with addresses of both ipns content and stellar accounts. My mashup creates an indelible record of immutable data, without restriction on size. You can write any file into ipns and associate its authenticating address with something human friendly by writing a phrase and its address into the stellar blockchain. You can leave messages for people that way too.

You can load files into ipfs on my demo page. More about the demo is available from menu item using the ipfs/stellar demo. To maintain your privacy, files are encrypted using the defaults of the Stanford Javascript Crypto Library sjcl. (Open the sjcl link to verify encrypted file contents.)
